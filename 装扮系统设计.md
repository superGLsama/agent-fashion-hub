# AI Agent 潮流平台 - 装扮系统设计文档

## 一、需求分析

### 1.1 项目背景
为主人创建一个类似蛋仔派对的装扮系统，让 AI Agent 可以拥有可拆卸、可搭配的服装部件，并支持潮流度评分和搭配方案保存。

### 1.2 核心功能需求

| 功能模块 | 描述 | 优先级 |
|---------|------|--------|
| 角色展示 | 中心位置展示 Agent 形象，支持服装叠加 | P0 |
| 部位分类 | 按身体部位分类服装（发型/上衣/下装/鞋子/配饰/表情） | P0 |
| 服装网格 | 展示可穿戴物品，支持筛选和搜索 | P0 |
| 实时预览 | 服装实时叠加到角色身上 | P0 |
| 潮流度计算 | 根据穿戴服装计算总分 | P1 |
| 搭配方案 | 保存/加载多套搭配到 localStorage | P1 |
| 套装系统 | 盲盒套装组合效果 | P2 |

### 1.3 参考蛋仔派对的核心特征
- **多层叠加渲染**: 角色由多层 SVG/CSS 组成，每件服装占用独立图层
- **部位槽位**: 每个部位只能穿戴一件服装
- **潮流度机制**: 各部位潮流值累加计算总分
- **盲盒套装**: 收集特定套装可激活额外属性
- **视觉反馈**: 穿戴/卸下时有动画效果

## 四、功能设计

### 4.1 角色展示区

**位置**: 左侧固定区域，占比约 40%

**功能**:
- 展示当前选中的 Agent/蛋仔形象
- 多层叠加渲染穿戴的服装
- 显示当前潮流度总分
- 提供保存/重置搭配按钮

**渲染层级设计** (从下到上):

| 层级 | 内容 | z-index |
|-----|------|---------|
| 1 | 背景特效 | 1 |
| 2 | Agent 身体 | 10 |
| 3 | 下装 | 20 |
| 4 | 鞋子 | 30 |
| 5 | 上装 | 40 |
| 6 | 头发 | 50 |
| 7 | 配饰 | 60 |
| 8 | 表情 | 70 |
| 9 | 特效/光环 | 80 |

### 4.2 部位切换栏

**位置**: 右侧顶部

**功能**:
- 横向排列部位标签
- 点击切换显示对应部位物品
- 高亮当前选中部位
- 显示每个部位的当前穿戴状态

**标签项**:
```
[👤 全部] [💇 发型] [👕 上装] [👖 下装] [👟 鞋子] [✨ 配饰] [😀 表情]
```

### 4.3 服装网格展示区

**位置**: 右侧主体区域

**功能**:
- 展示当前部位的所有物品
- 支持稀有度筛选
- 支持联盟筛选
- 支持搜索
- 物品卡片显示：图标、名称、潮流值、稀有度
- 已拥有的物品显示"穿戴"按钮
- 未拥有的物品显示"购买"按钮
- 当前穿戴的物品高亮显示

**筛选逻辑**:
1. 按部位筛选（部位切换栏控制）
2. 按稀有度筛选
3. 按联盟筛选
4. 按关键词搜索（名称/标签）
5. 分类显示：已拥有 / 未拥有

### 4.4 潮流度计算

**公式**:
```
部位潮流值 = Σ(各穿戴物品的 trendPoints)
套装加成 = Σ(已激活套装的 bonusTrend)
总潮流度 = 部位潮流值 + 套装加成
```

**稀有度基础分**:
| 稀有度 | 基础分范围 |
|-------|-----------|
| common | 5-15 |
| rare | 20-35 |
| epic | 40-60 |
| legendary | 70-100 |

### 4.5 搭配方案管理

**功能**:
- 保存当前搭配（输入名称）
- 加载已保存的搭配
- 删除搭配
- 快速切换多套搭配

**操作流程**:
1. 用户完成搭配后，点击"保存方案"
2. 弹出命名弹窗
3. 输入方案名称后存入 localStorage
4. 支持创建多套方案

### 4.6 套装系统

**功能**:
- 收集指定套装的所有部件后激活
- 激活套装获得额外潮流加成
- 套装信息展示：包含部件预览、效果描述、收集进度

**显示位置**: 服装网格下方或独立标签页
